## What an end-to-end "drive traffic to seller LP" campaign needs

**Important:** The existing `facebook-lead-gen-ad` skill defaults to Meta native instant-form (form pre-filled on FB). The user explicitly wants a click-through to `seller.ryan-realty.com/`. This is a different campaign objective and different approval gate. The playbook's footnote §12 actually flags this exact case: *"Build a dedicated `/sell/fb-valuation` landing page if Matt wants a click-to-website funnel as Campaign #4 once Lead Ad performance is established."* — the seller LP at `seller.ryan-realty.com/` already fills that role.

**Ad creative spec** (per `facebook-lead-gen-ad/SKILL.md` §4 + playbook §3 transfers):
- **Format:** Static image per playbook §3 (2026 finding: static beats video for lead-gen by 144% per BAM split-test). 1080×1080 PNG/JPG, text overlay <20%.
- **Variants:** 3 (per playbook §3 — Variant A "specific stat", B "question framing", C "counter-intuitive insight"). Meta picks winner in week 1.
- **Copy guardrails:** banned-word filter is built into `scripts/build-fb-ad.mjs` (matches CLAUDE.md voice rules — no "stunning", "must-see", "dream home", em-dashes, etc.).
- **Brand assets:** the canonical hero is `design_system/ryan-realty/assets/hero/banner-820x312-facebook.jpg` (pre-cropped Old Mill drone). Broker headshot variant available at `design_system/ryan-realty/assets/team/matt-ryan.png` (transparent PNG).

**Targeting recommendation** (lifted from playbook §1 — Special Ad Category Housing locks age/gender/ZIP):
- Geo: Bend metro 25mi radius (covers Redmond, Sisters, Tumalo)
- Age: 18+ (forced by Housing category — cannot exclude under-25)
- Detailed targeting: **NONE** (per playbook §5 — 2026 broad-targeting beats interest-targeting post-iOS privacy)
- Custom Audiences:
  - **Exclusion:** "Ryan Realty FUB suppression" (6,000-row CSV from `export-fub-custom-audience.mjs --mode suppression`) on every ad set
  - **Lookalike:** 1% Bend metro LAL seeded from FUB past sellers (4,964 rows from `--mode lookalike-seed`) → Campaign 2 (33% share)
  - **Website Custom Audience:** visitors to `/lp/seller-home-value` (and legacy `/sell*` paths) in last 30 days who didn't convert → Campaign 3 retargeting (17% share)
- Placements: Mobile only, FB + IG Feed/Stories/Reels. **Skip Audience Network, Messenger, in-stream video** (playbook §4 — low-quality leads).

**Budget + bidding starter:**
- $60/day CBO total across 3 campaigns (Cold $30 / LAL $20 / Retargeting $10) = ~$1,800/mo (playbook §1)
- Below $20/day per ad set there's no statistical signal in 7 days. Above $100/day, switch ABO.
- Bid: Lowest cost for first 7 days; after Meta exits learning phase (50 conversions), Campaign 1 switches to Cost cap at 1.5× desired CPL.

**Tracking (Pixel + CAPI + UTM strategy for the LP):**
- **Locked UTM convention** (playbook §7):
  - `utm_source=facebook`
  - `utm_medium=paid_social`
  - `utm_campaign={campaign-slug}` (e.g. `seller-cold-2026q2`)
  - `utm_content={variant-letter}` (e.g. `variant-a`)
- Dashboard attribution already counts visits matching `utm_source=facebook` OR `fbclid=` OR known FB/IG referrer (`app/actions/dashboard.ts` — `sellerVisitsFromFacebook30d`).
- **GAP — CAPI CORS:** `app/api/meta-capi/route.ts` lines 8–11 only whitelist `ryan-realty.com` + `www.ryan-realty.com`. The seller LP at `seller.ryan-realty.com` will fail cross-origin CAPI if the LP form ever calls the API from the subdomain. **Need to add `https://seller.ryan-realty.com` to `CORS_ALLOWED_ORIGINS` before launch** (or confirm the middleware rewrite serves it as same-origin — `middleware.ts` rewrites `seller.ryan-realty.com/` → `/lp/seller-home-value` so the browser sees `seller.ryan-realty.com` as origin but the request lands on the main app; cross-origin is required).
- Pixel + GA4 already fire on every page view via `PageViewTracker.tsx` after consent.

**Publishing surface — direct-to-LP, NOT instant form:**
- Campaign objective: **`OUTCOME_TRAFFIC`** or **`OUTCOME_SALES`/Conversions** with conversion event = `Lead` (matches CAPI Lead events from the LP form). NOT `LEAD_GENERATION` (which forces native FB form).
- Destination URL: `https://seller.ryan-realty.com/?utm_source=facebook&utm_medium=paid_social&utm_campaign={slug}&utm_content={variant}`
- CTA button: `LEARN_MORE` (better for traffic) or `GET_QUOTE` (signals what the LP delivers).
- This is **different from** what `facebook-lead-gen-ad/SKILL.md` documents — that skill assumes instant-form on Meta. The skill needs a sibling or an extension.

1. **No "click-to-website" Meta producer exists.** `facebook-lead-gen-ad/SKILL.md` is locked to native lead form. `ops-meta-ads/SKILL.md` only adjusts existing campaigns. The "create new campaign" action_type `ops:meta_ads_new_campaign` is referenced as "pending build" in `ops-meta-ads/SKILL.md` §1. **A new producer is needed** — call it `facebook-traffic-ad` or extend `facebook-lead-gen-ad` with a `campaign_mode: 'traffic'` branch.
2. **`scripts/create-fb-ad.mjs` is hard-coded to LEAD_GENERATION objective.** Would need a `--objective traffic` flag and a `--destination-url` arg.

**Asset gaps:**
- 3 static image variants per playbook §3 NOT pre-rendered. Would need either Canva pulls or the `flyer-design` producer (which is for printable flyers — wrong tool). Quickest path: human-curated Bend exterior photos from MLS/asset library + minimal text overlay rendered locally.

- Privacy URL works (`https://ryanrealty.vercel.app/privacy` already live per playbook §5).

**Manual steps that can't be code:**
1. Meta Ad Account access — `META_AD_ACCOUNT_ID` is set in production env, but the human running the launch needs Ads Manager access. Payment method must be on file on the ad account.

3. Lead form (if running Lead-Gen variant in parallel) — 5-minute manual create in Ads Manager unless we automate via `POST /v18.0/{page-id}/leadgen_forms`.
4. Webhook subscription — Meta App Dashboard → Webhooks → Page → `leadgen` field subscribed (likely already done; verify before launch).
5. **CAPI CORS update** — add `seller.ryan-realty.com` to `CORS_ALLOWED_ORIGINS` in `app/api/meta-capi/route.ts`. Same-origin if middleware rewrite handles it; confirm by hitting the LP and checking the request origin in dev tools.
